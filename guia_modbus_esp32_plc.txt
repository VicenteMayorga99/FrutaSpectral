GUIA RAPIDA - ESP32 ENC28J60 COMO SERVIDOR MODBUS TCP
=====================================================

Objetivo
--------
La ESP32 publica por Ethernet el valor procesado del sensor:

  promedioVisibleNm

Ese valor queda disponible para el PLC mediante Modbus TCP.


Datos de red de la ESP32
------------------------
Modulo Ethernet: ENC28J60
Modo IP: fija

  IP ESP32:      192.168.1.77
  Mascara:       255.255.255.0
  Gateway:       192.168.1.1
  Puerto Modbus: 502
  Unit ID:       1

El PLC y la ESP32 deben estar en la misma red IP o tener una ruta valida.
Ejemplo compatible:

  ESP32: 192.168.1.77
  PLC:   192.168.1.xxx
  Mascara de ambos: 255.255.255.0


Como "reconocer" la ESP32 desde el PLC
--------------------------------------
Modbus TCP no funciona como USB ni como PROFINET con descubrimiento automatico.
El PLC no va a "encontrar" la ESP32 por nombre.

Debes configurar al PLC para conectarse a:

  IP remota:     192.168.1.77
  Puerto remoto: 502
  Protocolo:     Modbus TCP cliente/master
  Unit ID:       1

La ESP32 queda trabajando como:

  Modbus TCP server/slave

El PLC debe trabajar como:

  Modbus TCP client/master


Mapa de registros Modbus
------------------------
La ESP32 responde lectura de holding registers con funcion Modbus 03.
Tambien acepta funcion 04 para facilitar pruebas, pero se recomienda usar 03.

Registro principal:

  Holding register 40001
  Direccion Modbus base 0
  Nombre: promedioVisibleNm x 100
  Tipo: entero sin signo de 16 bits
  Escala: dividir por 100.0 en el PLC

Ejemplo:

  Si el PLC lee 59023:
    promedioVisibleNm = 590.23 nm

Registro auxiliar:

  Holding register 40002
  Direccion Modbus base 1
  Nombre: dato valido
  Tipo: entero sin signo de 16 bits
  Valores:
    0 = aun no hay dato procesado
    1 = ya hay dato valido del sensor


Ojo con la numeracion 40001
---------------------------
Algunos programas o PLC piden el registro como 40001.
Otros piden la direccion interna base cero.

Para este codigo:

  40001 equivale a direccion 0
  40002 equivale a direccion 1

Si el PLC pregunta por "Start address", normalmente usa:

  Start address = 0
  Quantity      = 1

Si el PLC pregunta por "Register number", puede pedir:

  Register = 40001


Configuracion esperada en el PLC
--------------------------------
En el PLC Siemens S7-400 con CP 443-1, la comunicacion Modbus TCP no es lo
mismo que PROFINET ni S7 Communication.

Necesitas usar una de estas opciones:

1. Bloques/libreria Modbus TCP cliente para Siemens.
2. Bloques de comunicacion TCP abierta y armar Modbus TCP manualmente.
3. Un gateway o software intermedio que lea Modbus TCP.

La configuracion conceptual es:

  Tipo de conexion: TCP client
  Equipo remoto:   192.168.1.77
  Puerto remoto:   502
  Funcion Modbus:  03 Read Holding Registers
  Unit ID:         1
  Start address:   0
  Quantity:        1 o 2


Pruebas antes de usar el PLC
----------------------------
1. Conectar ESP32 + ENC28J60 a la red RJ45.
2. Confirmar por monitor serial que aparece:

   OK: ENC28J60 listo con IP fija.
   IP: 192.168.1.77
   Modbus TCP: puerto 502, Unit ID 1

3. Desde un PC en la misma red, probar en navegador:

   http://192.168.1.77

   Esto no es Modbus, solo sirve para confirmar que la ESP32 responde por red.

4. Probar con un cliente Modbus TCP de PC:

   IP:       192.168.1.77
   Port:     502
   Unit ID:  1
   Function: 03
   Address:  0
   Quantity: 1

   Debe devolver el valor promedioVisibleNm multiplicado por 100.


Diagnostico rapido
------------------
Si el PLC no comunica:

- Verificar que el RJ45 tenga link.
- Verificar que PLC y ESP32 esten en la misma subred.
- Hacer ping a 192.168.1.77 si la red/herramienta lo permite.
- Abrir http://192.168.1.77 desde un PC.
- Verificar que el PLC este usando Modbus TCP cliente, no PROFINET ni S7.
- Revisar si el bloque Siemens usa direccion 0 o registro 40001.
- Confirmar que el puerto remoto sea 502.
- Confirmar Unit ID = 1.


Dato entregado
--------------
El unico dato del sensor publicado por Modbus en esta version es:

  promedioVisibleNm

Sale desde ProcesamientoDatos.ino, donde se calcula como promedio ponderado
de las bandas visibles F4-F8.
